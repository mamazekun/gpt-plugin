# Chat-Navigator

一个面向长对话场景的 Chrome Extension（Manifest V3），用于在 AI 对话页面中提取用户问题，并在右侧生成问题导航面板，方便快速回到历史提问位置。

当前版本是可运行的 MVP，重点解决两件事：

- 对话很长时，快速定位“我之前问过什么”
- 页面持续流式更新时，导航列表仍能稳定同步

## 功能概览

- 支持站点
  - `chatgpt.com`
  - `chat.openai.com`
  - `gemini.google.com`
  - `bard.google.com`
- 自动识别用户消息并生成问题列表
- 在页面右侧注入悬浮触发条和可展开导航面板
- 支持按关键词过滤历史问题
- 搜索框右侧提供主题切换按钮（浅色 / 深色）
- 点击问题后平滑滚动到对应消息位置
- 根据当前滚动位置自动高亮当前问题
- 支持基于历史问题生成“追问”草稿并回填输入框
- 使用 `MutationObserver` 监听对话变化，优先增量刷新，必要时回退到全量刷新
- 当问题数量小于等于 1 条时自动隐藏导航

## 交互说明

插件注入后，页面右侧会出现一个窄触发条：

- 鼠标移入触发条或面板时展开
- 鼠标移出后延迟收起
- 面板顶部提供搜索框，右侧有主题切换图标按钮
- 每条问题右侧提供 `追问` 按钮
- 问题过多时，列表区域自动滚动
- 主题切换状态会记忆到本地（`localStorage`），刷新页面后保持上次选择

## 追问能力

点击某条问题右侧的 `追问` 按钮后：

- 如果该问题是当前最新一条用户问题，保持当前输入框内容不变
- 如果该问题是历史问题，将按以下模板回填输入框

```text
回到刚才的问题：
{历史问题}

在这个基础上，我想进一步追问：
{当前输入框内容}
```

为避免重复基于同一条问题连续追问，已经追问过的问题会被禁用；如果问题本身已经是追问模板内容，也不会再次允许追问。

## Gemini 适配细节

Gemini 页面做了额外清洗与兼容处理：

- 清理标题中的 `你说`、`You said`、`You:` 等前缀
- 尽量合并附件名和正文，避免导航标题只有附件信息或正文信息
- 对 Gemini 的 contenteditable 输入框使用专门的读写逻辑，保证回填后页面状态同步

## 技术实现

### 目录结构

- `manifest.json`：扩展入口与内容脚本注入配置
- `src/content/index.ts`：初始化、滚动同步、DOM 监听、问题刷新
- `src/content/dom-parser.ts`：站点适配、用户消息提取、输入框读写
- `src/content/navigation.ts`：问题跳转与当前激活项计算
- `src/content/panel.ts`：右侧悬浮面板的创建、搜索、列表渲染与交互
- `src/content/follow-up.ts`：追问能力可用性判断
- `src/content/styles.css`：面板样式
- `src/utils/types.ts`：公共类型定义

### 刷新策略

为兼顾性能和稳定性，当前实现采用两层刷新策略：

- 文本变化或新增节点时，优先只更新受影响的问题项
- 出现删除节点等结构变化时，执行全量刷新

另外，站内路由切换时对话根节点可能被替换，因此入口会定时重新确认观察目标并重建监听。

## 本地开发

### 安装依赖

```bash
npm install
```

### 类型检查

```bash
npm run check
```

### 构建

```bash
npm run build
```

构建产物输出到 `dist/content/index.js`。

## 加载到 Chrome

1. 打开 `chrome://extensions`
2. 开启右上角 `Developer mode`
3. 点击 `Load unpacked`
4. 选择当前项目根目录
5. 打开 ChatGPT 或 Gemini 对话页验证是否正常注入

补充说明：

- `manifest.json` 直接引用 `src/content/styles.css` 和构建后的 `dist/content/index.js`
- 修改 TypeScript 后需要重新执行 `npm run build`
- 修改 CSS 后通常刷新扩展和页面即可看到效果

## 当前限制

- 当前只适配 ChatGPT 和 Gemini，未覆盖 Claude、豆包等其他对话站点
- 用户消息识别依赖页面 DOM 结构与选择器，站点改版后可能需要同步调整
- 输入框回填依赖目标站点的前端事件机制，编辑器结构变化后可能需要更新适配逻辑
- Gemini 附件提取依赖可见节点和属性选择器，不保证覆盖所有上传组件形态
- 当前没有背景脚本、配置页、持久化存储、快捷键和跨页面同步能力

## 后续方向

如果继续扩展，这个项目比较适合往下面几类能力演进：

- 新站点适配：Claude、Perplexity、豆包、Kimi 等
- 更强导航：分组、时间排序、固定问题、最近访问记录
- 可配置能力：面板位置、展开方式、标题长度、快捷键
- 数据能力：本地持久化、会话级缓存、导出历史问题

README 中的功能说明应以 `src/content/` 下的实际实现为准。
